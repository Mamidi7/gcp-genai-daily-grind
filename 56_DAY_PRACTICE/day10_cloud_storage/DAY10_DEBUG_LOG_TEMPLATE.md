# Day 10 Debug Log Template

Use this every time a manual GCP step, FastAPI request, or Gemini call fails.

## Entry 1

### Topic
- Example: GCS bucket creation

### Symptom
- What exactly failed?

### Trigger
- What action caused it?

### Root Cause
- Why did it fail?

### Fix
- What changed to solve it?

### Prevention
- What rule/check would stop this from happening again?

### Impact
- What was blocked or improved?

### Interview Framing
- How would I explain this as real debugging work?

## STAR Conversion Template

Situation:
- What system were you working on?

Task:
- What did you need to make work or improve?

Action:
- What did you inspect, change, or validate?

Result:
- What became more reliable, clear, or fast?

## Good examples of day 10 debug topics

- bucket naming collision
- permission denied on object access
- wrong storage class choice
- missing env var
- invalid FastAPI payload
- upstream model timeout
- wrong error code mapping
- missing request tracing
