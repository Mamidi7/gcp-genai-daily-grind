# Tool Usage Matrix (Free-Only, Interview-First)

Use each tool only when it produces a concrete artifact.

| Tool | Use It For | Trigger | Artifact Output |
|---|---|---|---|
| roadmap.sh | Skill gap planning | Start of week | Weekly gap list in daily artifact |
| explainshell.com | Bash flag understanding | Command confusion in debug | Command breakdown note |
| regex101.com | Regex validation | Parsing/log filters fail | Regex test screenshot + final regex |
| jsoncrack.com | JSON structure visualization | Complex API/LLM response | JSON flow diagram in portfolio assets |
| bundlephobia.com | Dependency size check | Before adding npm package | Package decision note with size |
| transform.tools | Format conversion | Need JSON/YAML/TS/Pydantic conversion | Converted schema snippet in notes |
| webhook.site | Webhook inspection | Testing callbacks/events | Request/response evidence screenshot |
| devhints.io | Quick syntax refresh | During coding block | Cheat snippet used in daily note |
| httpie.io/app | API testing | New endpoint or bug reproduction | API request/response capture |
| caniuse.com | Browser support validation | Portfolio UI/CSS decision | Compatibility note in portfolio asset |
| squoosh.app | Image compression | Before adding screenshots | Before/after size comparison |
| carbon.now.sh | Code screenshot cards | Portfolio update day | Code card image |
| overapi.com | One-page command cheat sheet | Fast interview revision | Reference bullets in interview pack |
| shortcuts.design | Productivity shortcuts | Tool friction/repetition | Shortcut list in weekly notes |
| tinywow.com | File conversion fallback | PDF/image/doc conversion need | Converted file log entry |
| getfluently.app | Speaking practice (free mode) | Mock interview day | Speaking feedback summary |

## Priority Rule

If time is short, use in this order:

1. httpie.io/app
2. webhook.site
3. regex101.com
4. roadmap.sh
5. carbon.now.sh

## Fallback Rule

If a tool is temporarily unavailable, do not block progress.
Use a practical fallback and still produce the artifact.

- explainshell.com -> `man` pages or tldr pages.
- getfluently.app -> self-recorded mock answer + manual feedback notes.
