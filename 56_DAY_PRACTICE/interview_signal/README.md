# Interview Signal Engine (14-Day Sprint)

Purpose:
Turn daily prep into interview-proof artifacts using free web tools, without breaking coding momentum.

## Sprint Rules

- Daily tool time cap: 30 minutes.
- Primary goal: interview artifacts.
- Use only free features.
- Every day must end with one commit.

## Folder Map

- `daily_artifacts/` — one markdown file per day using the daily template.
- `interview_packs/` — 30s, 90s STAR, and 3-minute technical walkthrough packs.
- `portfolio_assets/` — diagrams, screenshots, compressed images, and supporting notes.

## 14-Day Flow

1. Pick one active coding topic from your daily practice.
2. Use the tool cadence from `14_DAY_EXECUTION_TRACKER.md`.
3. Capture failure path using `DEBUG_JOURNAL_TEMPLATE.md`.
4. Convert work into interview story using `INTERVIEW_PACK_TEMPLATE.md`.
5. Commit before day ends.

## Non-Negotiable Output Contract

Each daily artifact must include:

- `daily_topic`
- `what_built`
- `what_failed`
- `fix_applied`
- `prevention`
- `interview_30s`
- `interview_90s_star`
- `interview_3min`
- `tools_used`

## Start Today

1. Copy `DAILY_ARTIFACT_TEMPLATE.md` into `daily_artifacts/dayXX_YYYY-MM-DD.md`.
2. Follow today's row in `14_DAY_EXECUTION_TRACKER.md`.
3. Create or update one interview pack in `interview_packs/`.
4. Commit with: `dayXX: interview signal artifact + debug evidence`.
