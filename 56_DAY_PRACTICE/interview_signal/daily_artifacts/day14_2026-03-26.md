## Top Interview Story — Day 13

**90-second STAR:**

> **Situation:** Raw Pydantic 422 responses crashed our exception handler because `exc.errors()` embeds non-JSON-serializable Python objects.
>
> **Task:** Add strict input validation, normalize all error responses, enforce business rules at the API layer.
>
> **Action:**
> 1. Pydantic `Field()` constraints + `@field_validator` for custom logic
> 2. Custom `RequestValidationError` handler with `_sanitize()` — strips `ctx` sub-dicts before JSON serialization
> 3. `HTTPException` for reserved keyword violations (400)
> 4. Removed incorrect return type annotation `-> dict[str, str]` that caused `ResponseValidationError`
>
> **Result:** 13/13 tests pass. Blank input → 422, oversized → 422, reserved keyword → 400. Clean JSON shapes every time. Handler never sees bad data.

**Key insight for interviewers:** Type hints in FastAPI aren't documentation — they trigger response validation. Never assume library error objects are JSON-safe.
