# Day 11 - 2026-03-21

## daily_topic

Files + JSON + Environment Variables with fail-fast validation

## what_built

- Loaded configuration from `.env` using `python-dotenv`.
- Added JSON runtime config loading from `config/runtime_profile.json`.
- Added safe config endpoint: `/config-summary` (no raw secrets or full local paths).
- Added validation endpoint: `/config-validate` (fails when `GCP_PROJECT_ID` is missing).

## what_failed

`/config-validate` returned `503` when `GCP_PROJECT_ID` was not set.

## fix_applied

Set `GCP_PROJECT_ID` in `.env` and validated config again.

## prevention

- Keep `.env.example` as the required contract.
- Use `/config-validate` before running app in new environments.
- Never hardcode project IDs or credentials in source code.

## interview_30s

## interview_3min

Input:
- Environment variables (`GCP_PROJECT_ID`, `GCP_LOCATION`, credentials path)
- Local JSON runtime file (`runtime_profile.json`)

Process:
1. App loads `.env`.
2. It reads JSON runtime metadata.
3. `/config-summary` reports only safe values (masked paths, flags, key list).
4. `/config-validate` checks critical env completeness.

Output:
- Reliable startup diagnostics.
- Explicit `503` error for missing required env var.

Failure mode:
- If `GCP_PROJECT_ID` is absent, validation fails immediately.

Verification:
- Unit tests for summary shape, JSON load, and missing env behavior.
