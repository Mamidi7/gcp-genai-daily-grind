Wrap every API call. Always set timeout. Validate HTTP status before parsing JSON.
import requests.exceptions

def fetch(url):
    try:
        r = requests.get(url, timeout=10)
        r.raise_for_status()
        return {"success": True, "data": r.json()}
    except requests.exceptions.ConnectionError:
        return {"success": False, "error": "Connection failed"}
    except requests.exceptions.Timeout:
        return {"success": False, "error": "Request timed out"}
    except requests.exceptions.HTTPError as e:
        return {"success": False, "error": f"HTTP error: {e}"}
⚡ Order: raise_for_status() BEFORE .json(). A 500 often returns HTML — .json() on HTML crashes with ValueError.

Common Mistakes

🔥 Direct .json() crash

A 500 error page is HTML, not JSON. Call raise_for_status() FIRST.

🔥 Missing safety check

If the prompt is blocked, candidates may be empty. Always check if result.get("candidates") first.

🔥 Forgetting [0]

candidates is a LIST. You NEED [0] to get the first one. Same for parts.

4. Common Mistakes

🔥 No timeout

Without timeout=10, a hanging server freezes your app forever.

🔥 .json() before raise_for_status()

500 returns HTML — .json() crashes with ValueError.

🔥 data= instead of json=

Form-encoded ≠ JSON. Use json= for REST APIs.

🔥 Bare except:

Catches everything including KeyboardInterrupt.

🔥 Hardcoded keys

Use os.environ.get(), never "AIza..." in source.

🔥 Ignoring status_code

A 404 returns valid JSON too — but it's an error.
