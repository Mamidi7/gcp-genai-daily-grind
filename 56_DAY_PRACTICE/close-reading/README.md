# Close Reading - SolveIt Style

> Learn deep by having conversations with your study materials

Inspired by Jeremy Howard's SolveIt approach + ForBo7/fastai-close-reading

---

## What is Close Reading?

Instead of passively reading, you:
1. **Read a section** (e.g., "Functions in Python")
2. **Ask clarifying questions** to the AI
3. **Explore rabbit holes** - deep dive into interesting parts
4. **Generate summaries** for continuity
5. **Create flashcards** for retention

---

## How to Use This Folder

### Structure
```
close-reading/
├── summaries/          # Auto-generated chapter summaries
├── conversations/     # Your Q&A with AI about each topic
├── questions/         # Comprehension questions
├── anki/              # Generated flashcards
└── rabbit-holes/      # Deep dives into interesting topics
```

### For Each Study Topic (e.g., Day 9 - Functions)

1. **Read the material** (your notes, README, notebook)
2. **Ask me questions** like:
   - "Explain *args in simple terms"
   - "Why use kwargs over regular parameters?"
   - "Give me a real-world example"
3. **I'll generate**:
   - Summary of what you learned
   - 3-5 comprehension questions
   - Anki flashcards

## Tips for Deep Learning

1. **One topic at a time** - Don't rush
2. **Ask "why"** - Don't accept surface answers
3. **Example-driven** - Always ask for code examples
4. **Mistake-focused** - Ask "what's the common mistake?"
5. **Practice out loud** - Say answers like interviews
