Telugu lo — Full Pipeline

Browser = Nuvvu. Phone pattukuni mana app open chesav. "Data kavali" ani request pampistav — ala. Browser leads the request.

Cloud Run = Gate keep la untadi. Nee request ostundi, adi chusukuni "sarigga undi, security clear" ani allow chestundi, tarvata correct worker ki redirect chestundi. Reverse proxy, autoscaling, HTTPS — idantha Cloud Run chestadi.

Gunicorn = Okadu manager la. Nee request vachindi, Gunicorn daggara ready ga 4-5 workers untaru. Evariki pani takkuvo vallaki ee request assign chestadu. Concurrency handle cheyadaniki — okadu busy ante inko worker untadi.

Flask App = Chef la. Request vachindi, specific route batti logic run chestadu. /health ante "ok" returns, / ante "Hello" returns. Pure application logic.

Application Logic

Three endpoints: / (hello metadata), /health (liveness), /info (dynamic config). Uses jsonify(). The if __name__ == "__main__" block runs only in dev; gunicorn imports the module in production.

# ===== 1. Imports ===== from flask import Flask, jsonify import os # ===== 2. App instance ===== app = Flask(__name__) # ===== 3. Routes ===== @app.route('/') def hello(): return jsonify({ "message": "Hello from Cloud Run!", "status": "deployed", "platform": "Google Cloud Run" }) @app.route('/health') def health(): return jsonify({"status": "healthy"}) @app.route('/info') def info(): return jsonify({ "app": "Flask on Cloud Run", "port": os.environ.get("PORT", 8080) }) # ===== 4. Dev entry point ===== if __name__ == "__main__": port = int(os.environ.get("PORT", 8080)) app.run(host="0.0.0.0", port=port)
Pattern: if __name__ == "__main__"
Runs only on python main.py. In production, gunicorn imports the module and this block is skipped.
Pattern: os.environ.get("PORT", 8080)
Cloud Run sets PORT dynamically. Must match app.run(), EXPOSE, and gunicorn --bind.
Dependencies

flask for routing + gunicorn for production concurrency. Flask's dev server is single-threaded — gunicorn forks workers.

flask>=2.0.0 gunicorn>=20.0.0
F
flask

Micro web framework. >=2.0.0 for async compatibility. ~200KB.

G
gunicorn

WSGI server with fork-based workers. Handles concurrent requests.

Common Mistake: Deploying without gunicorn — Flask's dev server handles one request at a time. Under Cloud Run's load balancer, the instance timeouts and restarts in a loop.
Container Recipe

python:3.11-slim (~120MB vs 1GB+). Layer caching: dependencies install before code copy — code-only changes rebuild in seconds.

# 1. Base image — minimal OS layer FROM python:3.11-slim # 2. Working directory WORKDIR /app # 3. Copy deps first (layer caching) COPY requirements.txt . # 4. Install — cached until requirements changes RUN pip install --no-cache-dir -r requirements.txt # 5. Copy the app code COPY . . # 6. Document the port EXPOSE 8080 # 7. Production entry point CMD ["gunicorn", "--bind", "0.0.0.0:8080", "main:app"]
Layer Caching
Docker caches each layer. By copying requirements.txt before code, pip install only re-runs when deps change — code-only builds in ~5s instead of ~60s.
CMD Format
Exec form CMD ["gunicorn", ...] forwards SIGTERM correctly. Shell form CMD gunicorn ... wraps in sh -c which traps signals.
Clean Builds

Without .dockerignore, venv, __pycache__, .git inflate the build context to 200MB+. Exclude dev artifacts.

__pycache__/ *.pyc .env .git .gitignore venv/ .venv/ .DS_Store *.egg-info/
Impact: A 5MB Flask app balloons to 200MB+ in build context without .dockerignore.

Common Mistakes + Fixes

Real errors from real Cloud Run deployments — symptom, root cause, fix.
1
Missing gunicorn

Symptom: Container crashes under load. Fix: CMD ["gunicorn", "--bind", "0.0.0.0:8080", "main:app"]

2
PORT mismatch

Symptom: Health checks fail. Fix: os.environ.get("PORT", 8080) everywhere.

3
No .dockerignore

Symptom: 2+ min builds. Fix: Add .dockerignore excluding dev artifacts.

4
127.0.0.1 vs 0.0.0.0

Symptom: Works locally, fails on Cloud Run. Fix: host="0.0.0.0"

5
Wrong CMD format

Symptom: Container exits immediately. Fix: Exec form, not shell form.

6
Layer order wrong

Symptom: Full pip reinstall on code change. Fix: COPY requirements.txt first.

Deployment — Local to Cloud Run

From python main.py to a live HTTPS endpoint.
Figure: 4 stages — Dev → Docker → Registry → Cloud Run
Telugu lo — Deployment Pipeline

Local Dev = Nuvvu kitchen lo biryani tayaru chestunnav la. Ingredients anni ready cheskuntav, taste test chestav. Alane, python main.py lo app ni local ga test chestav.

Docker = Biryani ni box lo pack chestunnav la. Container lo anni dependencies, code, config anni fix chestav. Ekkada theeskellina oke taste — same behavior.

Artifact Registry = Godown lo pettadam la. Docker image ni store chestadi. Tarvata Cloud Run deggara nunchi pull chestadi deploy cheyadaniki.

Cloud Run = Restaurant counter la. Customer request vachindi, image ni instant ga serve chestadi. 0 to 1000 auto-scale. Pay only when running.

Flow: Local Dev → Docker (pack) → Artifact Registry (store) → Cloud Run (serve) → Live at *.run.app
