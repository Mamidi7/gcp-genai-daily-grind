# main.py — Click a card above to switch files
from fastapi import FastAPI
from pydantic import BaseModel
import requests, os

app = FastAPI()

class Prompt(BaseModel):
    text: str

GEMINI_KEY = os.environ["GEMINI_API_KEY"]
MODEL = "gemini-2.0-flash-001"
BASE = "https://generativelanguage.googleapis.com/v1beta/models"

@app.get("/health")
def health():
    return {"status": "ok"}

@app.post("/chat")
def chat(body: Prompt):
    url = f"{BASE}/{MODEL}:generateContent?key={GEMINI_KEY}"
    payload = {"contents": [{"parts": [{"text": body.text}]}]}
    r = requests.post(url, json=payload, timeout=30)
    r.raise_for_status()
    result = r.json()
    answer = result["candidates"][0]["content"]["parts"][0]["text"]
    return {"response": answer}

Deployment Pipeline

Three commands. Watch each stage as your code travels from laptop to live URL.

📝 Stage 1: Your Code
You have three files: main.py (FastAPI app), requirements.txt (dependencies), and Dockerfile (build instructions). Run gcloud builds submit to send them to Cloud Build.
# Step 1: Build container
gcloud builds submit --tag us-central1-docker.pkg.dev/$PROJECT/app/main:latest

# Step 2: Deploy to Cloud Run
gcloud run deploy main \
  --image us-central1-docker.pkg.dev/$PROJECT/app/main:latest \
  --region us-central1 --allow-unauthenticated \
  --set-env-vars GEMINI_API_KEY=your_key_here

What Docker Actually Does

Each line in the Dockerfile is a step. Hover to see what it does.

💡 Think of it like this: Docker is a shipping container. Your app, Python, and all packages go inside. The container runs identically on your laptop, on your friend's Windows PC, and on Google's servers. No "but it works on my machine!" problems.

Common Mistakes

🔥 Port mismatch

Cloud Run sets PORT=8080. Your app MUST listen on whatever PORT says. If you hardcode 8000, health checks fail → revision killed.

port = int(os.getenv("PORT", 8080))

🔥 Hardcoded secrets

If you write API_KEY = "AIza..." in main.py, that key is baked into your Docker image forever. Use env vars.

🔥 No try/except

Gemini might fail. If you don't catch errors, FastAPI returns a 500 with no helpful message. Always wrap external calls.

🔥 Confusing layers

FastAPI ≠ deployment. Docker ≠ deployment. Cloud Run IS deployment. FastAPI runs locally. Docker packages it. Cloud Run hosts it.

🎯 3-minute technical walkthrough

FastAPI sits on Starlette (ASGI) and Pydantic. When a request hits, Uvicorn passes it to FastAPI, which matches the URL path to a route decorator. For POST with JSON, Pydantic auto-validates the body — missing fields return a 422 BEFORE your function runs. You never write manual validation.

For deployment: Cloud Build reads the Dockerfile. Each instruction is a cached layer — pip install only re-runs when requirements.txt changes. The image is stored in Artifact Registry. Cloud Run then pulls the image into a gVisor sandbox — a lightweight container runtime with kernel-level isolation. Cloud Run sets PORT=8080, pings /health every 5s, and scales from 0 to N instances based on traffic.

Key constraint: stateless. No filesystem persistence. For our Gemini proxy, that's fine — but databases or file uploads need Cloud Storage, which is Day 9.
