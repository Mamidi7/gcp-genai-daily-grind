# Daily Operating System

Use this file every day.

## Daily Order

```text
Mission
-> Action list
-> High-level concept
-> ASCII diagram
-> Block-wise why
-> Manual GCP/UI steps
-> Notebook work
-> Code / SQL / BigQuery
-> Forced failure
-> Debug record
-> Interview answer
-> Day closeout
-> Commit
```

## Daily Minimum

Before the day can count, produce:
- one note or concept summary
- one code snippet or SQL query
- one debug artifact
- one interview artifact
- one commit-ready change

## Stuck Mode

If the full build does not finish, do this:
- draw the diagram
- write the simple-English summary
- solve one SQL/BQ query or code snippet
- record one failure or failure hypothesis
- write one 30-second interview answer
- update `SESSION_STATE.md`
- update `DAILY_PROGRESS_LOG.md`

The streak stays alive only if visible proof exists.

## Recall Mode

At the start of each next day:
- answer 3 questions from yesterday
- redraw one small diagram
- say the 30-second explanation without notes

## Weekly Discipline

Every review day:
- revisit one debug story
- re-solve one SQL query
- re-explain one topic without notes
- clean one README or closeout artifact
