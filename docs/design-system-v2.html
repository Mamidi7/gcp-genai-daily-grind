08 Animations

Page Entrance

@keyframes pageIn { from { opacity: 0; transform: translateY(18px); } to { opacity: 1; transform: translateY(0); } }

Every <section> gets this. Trigger via IntersectionObserver adding class "in".

SVG Pipeline Flow

@keyframes pipeFlow { to { stroke-dashoffset: -18; } }

Animated arrow for data flow diagrams. Use with stroke-dasharray: 6 3.

Stamp Entrance

@keyframes stampIn { from { opacity: 0; transform: rotate(-12deg) scale(1.6); } to { opacity: .85; transform: rotate(3deg) scale(1); } }

Hero accent stamp. Typically positioned position: absolute; top: 0; right: 0; and animated with animation: stampIn .4s .6s cubic-bezier(.2,.8,.3,1.4) forwards;

Pulse (for indicators)

@keyframes pulse { 0%,100%{opacity:1;} 50%{opacity:.4;} }

For "live" indicators, loading states.

11 How to Use

1. Link the CSS file

<link rel="stylesheet" href="design-system.css">

Place this in your HTML <head>. The CSS file automatically imports Google Fonts via @import.

2. Add the dark mode toggle button (optional)

<button id="darkToggle" onclick=" var d=document.documentElement; if(d.getAttribute('data-theme')==='dark'){ d.removeAttribute('data-theme'); } else { d.setAttribute('data-theme','dark'); } ">Toggle dark</button>

The CSS automatically switches all colors when [data-theme="dark"] is set on <html>.

3. Use the component classes

All components use flat CSS class names — no BEM, no utilities framework. Just drop class="card", class="btn btn-accent", etc. See the sections above for every component's HTML pattern.

4. Section entrance animations

<section class="section">...</section> <script> var obs = new IntersectionObserver(function(e) { e.forEach(function(e) { if(e.isIntersecting) e.target.classList.add('in'); }); }, {threshold: 0.1}); document.querySelectorAll('.section').forEach(function(e) { obs.observe(e); }); </script>

5. Using in design system HTMLs

Since this repo uses HTML instead of Markdown for planning/communication (Thariq Shihipar pattern), place the CSS reference in your artifact's <head>. No build step required — it works from the filesystem.
