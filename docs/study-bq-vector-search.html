BigQuery Vector Search

Semantic search directly in SQL — embeddings, vector indexes, ANN search. High-signal interview topic for Applied AI roles.
Architecture

Embeddings

text-embedding-004

768-dimensional dense vectors. Google's latest model. Supports multilingual semantic precision. Call via ML.GENERATE_EMBEDDING() directly in SQL.

VECTOR INDEX

IVF algorithm

Approximate Nearest Neighbor index. Partitions vectors into clusters (num_lists). Without it, VECTOR_SEARCH falls back to brute force O(n) — slow on large data.

VECTOR_SEARCH

ANN query

Returns top-K nearest neighbors by distance. Supports COSINE, EUCLIDEAN, DOT_PRODUCT. No separate vector DB needed — all inside BigQuery.

SQL Implementation
-- Generate embeddings for your text data CREATE OR REPLACE TABLE my_dataset.documents AS SELECT doc_id, content, ML.GENERATE_EMBEDDING( MODEL my_dataset.embedding_model, (SELECT content AS text FROM my_dataset.raw_docs) ) AS embedding FROM my_dataset.raw_docs;
Model must reference text-embedding-004 via CREATE MODEL ... REMOTE WITH CONNECTION
-- Create vector index for ANN search CREATE VECTOR INDEX my_doc_index ON my_dataset.documents(embedding) OPTIONS( distance_type = 'COSINE', index_type = 'IVF', ivf_options = '{"num_lists": 100}' );
⏳ Index build is async — check with SELECT * FROM ML.INDEX_INFO(...)
-- Semantic search: find top-5 similar documents SELECT doc_id, content, distance FROM VECTOR_SEARCH( TABLE my_dataset.documents, 'embedding', ( SELECT embedding FROM ML.GENERATE_EMBEDDING( MODEL my_dataset.embedding_model, ('fraud detection in real-time transactions' AS text) ) ), top_k => 5, distance_type => 'COSINE' );

Interview Questions

30-Second Answer

quick
"BigQuery Vector Search lets you do semantic search directly in SQL. Generate embeddings with ML.GENERATE_EMBEDDING(), build a VECTOR INDEX, then call VECTOR_SEARCH() for ANN. No separate vector DB needed."

Common Mistake

avoid
❌ Calling VECTOR_SEARCH() without creating the index first. Without it, BigQuery brute-force scans all rows — O(n) and extremely slow on large tables.
