01

Syllogism — 5 Rules to Remember

5-8 questions. Easiest marks if you know the 5 rules. Venn diagrams not needed in exam — just rules.
The 5 Syllogism Rules

If you memorize only 5 things in reasoning, memorize these

  1. All + All = All (All A are B + All B are C = All A are C)
  2. All + Some = No Conclusion (All A are B + Some B are C = ?)
  3. Some + Some = No Conclusion (Some A are B + Some B are C = ?)
  4. All + No = No (All A are B + No B are C = No A are C)
  5. Some + No = Some Not (Some A are B + No B are C = Some A are not C)
REMEMBER: "All + Some" and "Some + Some" NEVER give a definite conclusion. If you see these in options, eliminate the "definite" answer. Only "possibility" answers are valid.
Example 1 — Definite Conclusion
Statements: All birds are animals. All animals are living things.
Pattern: All + All = All Conclusion: All birds are living things.
Answer: TRUE
Example 2 — No Conclusion
Statements: All boys are students. Some students are girls.
Pattern: All + Some = No Conclusion Conclusion: "All boys are girls" — FALSE (cannot say) Conclusion: "Some boys are girls" — FALSE (cannot say) Conclusion: "Some boys are not girls" — POSSIBLY TRUE (if stated as possibility)
Answer: No definite conclusion. Only "possibility" forms work.
Example 3 — Possibility Question
Statements: Some pens are books. No book is a table.
Pattern: Some + No = Some Not Conclusion: Some pens are not tables. (TRUE)
Answer: "Some pens are not tables" is TRUE
Common Trap: If conclusion says "Some A are B" and statements have "All A are B" — the conclusion is TRUE. "All" includes "Some." But reverse is NOT true. "Some" does not mean "All."
02

Inequality — Coded & Direct

3-5 questions. Takes 10 seconds each if you know the symbol logic.
> (Greater than)
A > B means A is greater than B
< (Less than)
A < B means A is less than B
>= (Greater than or equal)
A >= B means A is greater than or equal to B
<= (Less than or equal)
A <= B means A is less than or equal to B
= (Equal to)
A = B means A equals B
!= or <> (Not equal)
A != B means A is not equal to B
The Priority Rule

When comparing A and C through B, check the chain direction

  1. A > B and B > C = A > C (same direction = definite)
  2. A > B and B < C = No relation (opposite direction = nothing)
  3. A >= B and B >= C = A >= C (same direction with >=)
  4. A > B and B >= C = A > C (stronger sign wins)
  5. A >= B and B > C = No definite (>= includes equal, so A could equal B and B > C means A could equal something greater than C... actually A >= C still holds)
SHORTCUT: Same direction symbols = definite. Opposite directions = no conclusion. "=" is neutral and passes through.
Example
Statement: A > B >= C < D = E
A > B >= C → A > C (same-ish direction, > is stronger) A > B >= C < D → A vs D: opposite direction at C < D. No relation. C < D = E → C < E (D = E, so C < E) B >= C < D → B vs D: opposite directions. No relation.
A > C is TRUE. C < E is TRUE. A vs D = No relation.
Coded Inequality: They give you: P$Q means P>Q, P@Q means P
03

Coding-Decoding

3-5 questions. Pattern recognition. No deep logic needed.
3 Pattern Types

90% of coding questions follow one of these 3 patterns

  1. Letter shift: Each letter moved by fixed number. CAT → DBU (+1 each). CAT → FDW (+3 each).
  2. Reverse coding: Word is reversed first, then shifted. CAT → TAC → UBD.
  3. Position sum: Code is based on position in alphabet. A=1, B=2... CAT = 3+1+20 = 24. Or reverse: 24 = position sum.
TRICK: First check if letters are simply shifted. Count difference between first letters of word and code. If CAT=DBU, C→D = +1. Apply +1 to all. If works = solved.
Example — Letter Shift
In a code, BLIND is written as CMJOE. How is CIVIL written?
B → C (+1), L → M (+1), I → J (+1), N → O (+1), D → E (+1) Pattern: Each letter +1 CIVIL → D J W J M
Answer: DJWJM
Example — Reverse then Shift
In a code, RING is written as SJOH. Find the pattern.
R → S (+1), I → J (+1), N → O (+1), G → H (+1) Actually: simple +1 shift. RING reversed = GNIR, +1 = HOJS. But given code is SJOH. Wait — RING reversed = G N I R. +1 = H O J S. Not matching. Check: R→S, I→J, N→O, G→H. Direct +1. No reverse needed here.
Answer: Simple +1 letter shift. Always test direct shift first.
04

Blood Relations

2-3 questions. Family tree shorthand saves time.
P/Q's father
Write: Fa(P) or Fa(Q). One level up male.
P/Q's mother
Write: Mo(P). One level up female.
P/Q's brother
Same level, male. Same parents.
P/Q's sister
Same level, female. Same parents.
P/Q's son
One level down, male.
P/Q's daughter
One level down, female.
P/Q's wife
Spouse female. Same level.
P/Q's husband
Spouse male. Same level.
The Generation Method

Draw generations top to bottom. Don't draw full tree.

  1. Start with the person mentioned first. Put them in middle.
  2. "Father of" = one level UP. "Son of" = one level DOWN.
  3. "Brother of" = SAME level. "Wife of" = same level, opposite gender.
  4. Answer the question by counting levels and genders.
SHORTCUT: If they ask "how is X related to Y", trace from X to common ancestor to Y. Count: father/mother = up one, son/daughter = down one, brother/sister = same level.
Example
Pointing to a man, Seema said, "He is the son of my grandfather's only son." How is the man related to Seema?
"My grandfather's only son" = Seema's father (could be her father or uncle, but "only son" = father) "Son of my father" = Seema's brother
Answer: Brother
05

Direction & Distance

2-3 questions. Simple coordinate method.
The Coordinate Method

Treat starting point as (0,0). North = +Y, East = +X, South = -Y, West = -X.

  1. Start at (0, 0)
  2. 10 m North → (0, 10)
  3. Then 5 m East → (5, 10)
  4. Then 10 m South → (5, 0)
  5. Final position: (5, 0). Distance from start = 5 m East.
SHORTCUT: Same direction movements (N then S, E then W) cancel out. Only calculate net movement in each direction.
Example
A walks 10 m North, 5 m East, 10 m South, 5 m West. Where is he from start?
North 10 + South 10 = 0 (cancels) East 5 + West 5 = 0 (cancels) Net = (0, 0) = starting point
Answer: At the starting point
Distance formula: sqrt(X² + Y²) where X = net East-West, Y = net North-South
Turns: "Turns left" = anticlockwise. "Turns right" = clockwise. If facing North and turn left → facing West. If facing North and turn right → facing East. Use clock face: 12=N, 3=E, 6=S, 9=W.
06

Puzzles & Seating Arrangement

15-20 questions. The biggest section. Master the box method.
The Box Method

For linear arrangement (straight line)

  1. Draw empty boxes: _ _ _ _ _ _ _ (7 people = 7 boxes)
  2. Mark "North/South facing" — North = looking UP (toward higher position numbers), South = looking DOWN
  3. Read ALL clues before placing anyone. Find the "fixed position" clue first.
  4. Place the person with the MOST constraints first.
  5. Use "not" clues for elimination (A is NOT at position 3).
GOLDEN RULE: Never solve clues in order. Read all clues. Find the one that places someone exactly. Then work outward. "A sits at an extreme end" = place A first. "B sits third to the left of C" = relative position — keep flexible.
Circular Arrangement

For round table

  1. Draw a circle. Mark positions like clock: 12, 1, 2, 3... for 8 people.
  2. "Facing center" = all look inward. "Facing outward" = all look away.
  3. "Immediate left" in a circle = clockwise if facing center. Counter-clockwise if facing outward.
  4. Pick one person as anchor (the one with most clues). Place at 12 o'clock.
SHORTCUT for direction: If everyone faces center, "left" = clockwise. "Right" = counter-clockwise. If facing outward, reverse: "left" = counter-clockwise, "right" = clockwise.
Time Trap: If a puzzle takes more than 3 minutes, MARK it and move on. Come back if time permits. 1 puzzle = 5 questions. If you can't crack it in 3 min, you lose 5 marks AND waste time. Skip smartly.
07

Input-Output

2-3 questions. Machine rearrangement pattern.
The Pattern Hunt

Common patterns in input-output

  1. Ascending/Descending by word length: Shortest word first, then next shortest...
  2. Alphabetical order: Words arranged A to Z (or reverse)
  3. Vowels/Consonants separated: Words starting with vowels first, then consonants
  4. Numbers: Ascending or descending
  5. Sum of digits: Arrange by sum of digits in numbers
TRICK: Compare Input → Step 1 and Step 1 → Step 2. What moved? If only ONE element changed position between steps, it's a "single element per step" machine. If multiple changed, it's a batch rearrangement. Most bank exams use single element per step.
Example
Input: word 32 is 45 the 12 best 28
Step 1: 12 word 32 is 45 the best 28
Step 2: 12 best word 32 is 45 the 28
Step 1: smallest number (12) moved to first position Step 2: word starting with earliest alphabet ("best") moved to second position Pattern: Number (ascending) then Word (alphabetical), alternating
Pattern: Smallest number → earliest word → next number → next word...
