Word Embeddings

How computers turn words into numbers that carry meaning

Core Idea

Every word becomes a list of numbers (a vector). Words with similar meaning get similar numbers. "King" and "Queen" sit close in this number-space.

The Problem

Old methods treated words as isolated labels. "Bank" (river) and "Bank" (money) had the same code. Context was lost. Meaning was flat.

The Solution

Neural networks learn vectors by reading millions of sentences. Words appearing in similar contexts end up with similar vectors. Math on vectors = math on meaning.

How It Works

The trick is co-occurrence. A neural network slides a window across text. For each word, it collects its neighbors. Words that share neighbors get pushed together in vector space. Words that never appear together drift apart.

After training, each word is a point in ~300 to 768 dimensions. You cannot visualize all dimensions, but you can project them to 2D (what the canvas above does). The structure you see — clusters, directions, distances — is real.

Key property: The vector difference between "king" and "man" is approximately the same as the vector difference between "queen" and "woman". This means the direction "man -> woman" is the direction "king -> queen". That direction encodes the concept of gender.

Code in Practice

NumPy 
import numpy as np

# Simplified 5-dimensional embeddings
king   = np.array([0.8, 0.2, 0.9, 0.1, 0.3])
man    = np.array([0.7, 0.3, 0.1, 0.1, 0.2])
woman  = np.array([0.6, 0.4, 0.1, 0.8, 0.2])

# The famous equation: king - man + woman ≈ queen
result = king - man + woman
# result ≈ [0.7, 0.3, 0.9, 0.8, 0.3]  ->  "queen"

BigQuery Vector Search 
SELECT query.id, base.text, distance
FROM VECTOR_SEARCH(
    TABLE my_dataset.embeddings,
    'embedding_col',
    (SELECT embedding FROM my_dataset.embeddings WHERE id = 42),
    top_k => 5,
    distance_type => 'COSINE'
);

Deep Follow-Up

Q: Why does cosine similarity work better than Euclidean for embeddings?

Cosine similarity measures the angle between vectors, ignoring magnitude. Two documents with the same topic but different lengths will have vectors pointing in the same direction but with different lengths. Euclidean distance would penalize the longer document. Cosine focuses on orientation = meaning, not scale = frequency.

Another Deep Follow-Up

Q: What is the curse of dimensionality, and does it hurt embedding search?

In high dimensions, distances become uniform — every point is roughly equidistant. But embeddings are not random. They live on a low-dimensional manifold within the high-D space. Approximate nearest neighbor (ANN) indexes like ScaNN or FAISS exploit this structure. They partition the manifold, not the full space, so search stays fast even at 768 dimensions.
