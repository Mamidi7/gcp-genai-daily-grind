## Errors Debugged

- **Returning dict directly from Flask route** — Symptom: works in some setups but lacks proper Content-Type header. Fix: Always use `jsonify()` to return a proper Flask Response object. Root cause: Flask expects Response object, dict auto-conversion is not guaranteed.
- **Hardcoded PORT** — Symptom: container fails health check on Cloud Run. Fix: Use `os.getenv("PORT", 8080)` and bind to `0.0.0.0`.
- **Copy order in Dockerfile** — Symptom: every code change triggers full pip reinstall (slow builds). Fix: Copy requirements.txt before code to leverage Docker layer caching.
