The 3 Phases of an FDE

Audit → Evals → Deployment. The entire job in three verbs.

PHASE 1
🔍 Audit
You're on-site with the client, mapping processes and workflows across teams. Two weeks with rev ops, one week with procurement, a full month with finance. You learn what their job looks like, where the bottlenecks are, and where AI agents could deliver value. Equally important: you figure out what NOT to automate.
On-site Workflow Mapping SCOPE Prototype
FDE's 3 Principles for What to Automate
1. Rules + Variable Inputs = Agent. Workflow can be distilled into rules but inputs are different (email vs PDF vs image) + involves calling tools → put an agent in.
2. Rules + Predictable Inputs = Code. If both are predictable, code is faster and cheaper. Don't over-engineer.
3. Needs Pattern Recognition = Keep Human. Domain expertise that can't be distilled → leave it manual.
PHASE 2
📊 Evals
If a customer is spending millions on an AI deployment, they need to know it's working. A good eval doesn't just check final answer correctness — it verifies the AI is thinking like a human would. Trace the human's steps and grade the AI on each one. Start with 20 golden examples of "what great looks like" and measure everything against them.
Golden Dataset Step-by-step Grading ROI Proof
PHASE 3
🚀 Deployment
Avoid large-scale data migrations. Instead, build APIs over existing data layers (SharePoint, databases) and place a model as an orchestrator on top. Create a sandbox in the company's infra for safe testing. Start small: one workflow, get it working, then layer on capabilities. Start with read-only (agent catches bugs, writes tickets), only if that works give it write permissions (push PRs).

30-Day Roadmap to FDE

4 checkpoints. Build these skills and your portfolio speaks for itself.

CHECKPOINT 1
Agent Fundamentals
Day 1-7
  • Agent loop: prompt → model → response → next step
  • Tool calling (API call + web search)
  • Guardrails: input validation, max-step limit, output filtering
  • Context window vs external memory
  • Audit trail: log every prompt, tool call, response
CHECKPOINT 2
Demo to Production
Day 8-14
  • Structured outputs (always return JSON)
  • What breaks when demo goes to prod
  • Checkpointing: save state every N steps
  • Error handling patterns
CHECKPOINT 3
Production Hardening
Day 15-21
  • Retry logic + exponential backoff (1s→2s→4s→8s→16s cap)
  • Cost optimization: cheaper models per subtask
  • Golden dataset for evals (20 real queries, labeled)
  • Multi-agent pipelines + parallel architectures
CHECKPOINT 4
Communication
Day 22-30
  • Review everything, communicate out loud
  • Tie everything to business metrics
  • Portfolio walkthrough: tech stack + iterations + outcomes
  • Explain to a non-technical VP in plain English
Key Insight
Lack of communication ability is a deal-breaker for the FDE role. If you can't explain what AI can and can't do to a non-technical decision maker, there won't be a deployment. The portfolio is table stakes — how you talk about it is the differentiator.

Interview Prep

Three depths — 30 seconds, 90 seconds, 3 minutes.

⏱️ 30-Second Answer ▼

Forward Deployed Engineers bridge the gap between AI capabilities and real business problems. We go on-site with customers, map their workflows, identify where AI agents create value (and where they don't), build evaluations that prove ROI, and deploy incrementally — starting with the smallest unit of autonomy and layering up. It's the hottest role in tech because intelligence is becoming a commodity, so competitive advantage comes from how and where you deploy it.

⏱️ 90-Second STAR Answer ▼

Situation: Our team needed to deploy an AI-powered health records pipeline for a clinic that stored data across PDFs, spreadsheets, and legacy systems.

Task: Build an automated pipeline that extracts lab values and trends, without migrating any existing data.

Action: I followed the FDE playbook — audited the workflow (PDF parsing bottleneck identified), built a golden dataset of 20 labeled records for evals, deployed a FastAPI layer over existing files (no migration), started with read-only extraction, verified accuracy, then layered on trend analysis.

Result: Lab extraction accuracy improved from manual errors to consistent structured output. Pipeline cost $2/day on Gemini API, zero infrastructure migration needed. Audit trail logged every step for debugging.

⏱️ 3-Minute Technical Walkthrough ▼

The FDE role has three phases — let's walk through each with a real example.


Phase 1: Audit. I sat with the clinic team and mapped their data workflow. PDF reports come in, nurses manually enter lab values into spreadsheets. Bottleneck: PDF parsing consumes 40 min/day. Two rules: (a) if the report format varies (different labs = different layouts), use an agent; (b) if the decision requires medical judgment (is this trend dangerous?), keep a human in the loop. Result: automate extraction, flag anomalies for human review.


Phase 2: Evals. I built a golden dataset — 20 PDFs where I hand-extracted the correct values. The eval checks: did the agent find the right field? Did it handle missing values correctly? Did it flag uncertainty? Each step gets graded. Executives saw the numbers and approved deployment.


Phase 3: Deployment. No database migration. FastAPI endpoints over the existing folder structure. Agent loop: receive PDF → Gemini 2.5 Flash extracts → structured JSON returned → appended to a daily log. Started small: extraction only (read), verified for 2 weeks, then layered on "detect worsening trends" (write flags). Cost: $0.002 per PDF on Gemini, ~100 PDFs/day = $0.20/day. 100x cheaper than manual entry with zero data migration risk.


Key learning: The eval phase is what executives care about most. If you can't prove ROI, you won't get the deployment. That's why golden datasets and step-by-step grading matter more than the agent itself.
