01

Syllogism — 5 Rules = 5-8 Marks

The easiest high-weight topic. Memorize 5 rules. Never draw Venn diagrams in exam.
The 5 Rules (Memorize These)

Every syllogism question uses one of these 5 patterns

  1. All + All = All — All A are B + All B are C = All A are C
  2. All + Some = No Conclusion — All A are B + Some B are C = Cannot say anything definite
  3. Some + Some = No Conclusion — Some A are B + Some B are C = Cannot say anything definite
  4. All + No = No — All A are B + No B are C = No A are C
  5. Some + No = Some Not — Some A are B + No B are C = Some A are not C
REMEMBER: "All + Some" and "Some + Some" NEVER give a definite conclusion. If options have definite statements (All/Some/No), they are WRONG. Only "possibility" answers work for these combinations.
Example 1 — Definite (All+All)
Statements: All pens are books. All books are copies.
Conclusions: I. All pens are copies. II. Some copies are pens.
Pattern: All + All = All I. All pens are copies → TRUE (direct from rule) II. Some copies are pens → TRUE ("All" includes "Some")
Answer: Both I and II follow
Example 2 — No Conclusion (All+Some)
Statements: All dogs are cats. Some cats are rats.
Conclusions: I. All dogs are rats. II. Some dogs are rats.
Pattern: All + Some = No Conclusion I. All dogs are rats → CANNOT SAY (no conclusion possible) II. Some dogs are rats → CANNOT SAY
Answer: Neither I nor II follows
Example 3 — Some+No
Statements: Some teachers are doctors. No doctor is a lawyer.
Conclusion: Some teachers are not lawyers.
Pattern: Some + No = Some Not Some teachers are doctors + No doctor is lawyer = Some teachers are not lawyers
Answer: Conclusion follows
Example 4 — Possibility Question (Common in RBI)
Statements: Some A are B. All B are C.
Conclusions: I. All A being C is a possibility. II. Some C are A.
Pattern: Some + All = Some (Some A are C) I. "All A being C" — Since Some A are C, it IS possible that ALL are C. TRUE. II. Some C are A — Since Some A are C, reverse is also true. TRUE.
Answer: Both follow
Common Trap: If conclusion says "Some A are B" and you know "All A are B" — the conclusion is TRUE because "All" includes "Some." But reverse is FALSE. "Some A are B" does NOT mean "All A are B."

Inequality — Coded & Direct

3-5 questions. 15 seconds each with the direction rule.
The Direction Rule

Same direction = definite. Opposite direction = no conclusion.

  1. A > B > C → A > C (same direction, definite)
  2. A > B < C → A vs C = No relation (opposite direction)
  3. A >= B >= C → A >= C (same direction)
  4. A > B >= C > D → A > D (chain: >, >=, > — all same-ish direction)
  5. A = B > C → A > C (= passes through, then >)
  6. A > B = C < D → A vs D: opposite at B > ... < D. No relation.
SHORTCUT: Look for the "pivot" where direction changes. If all symbols between two variables point the same way → definite relation. If any symbol points opposite → no relation. "=" is neutral.
Example — Complex Chain
Statements: P >= Q = R > S < T = U
P >= Q = R > S → P > S (all same direction from P to S) P vs T: P > S < T → opposite at S < T. No relation. R vs U: R > S < T = U → opposite at S < T. No relation. Q vs S: Q = R > S → Q > S (definite)
P > S is TRUE. P vs T = No relation. R vs U = No relation. Q > S is TRUE.
Coded Inequality

They give symbols like $, @, # — decode first, then apply same rules

  1. Write down the decoding: A $ B means A > B, A @ B means A < B, etc.
  2. Replace symbols with actual inequality signs in the statement
  3. Apply the direction rule exactly as above
TRICK: Write the decoded version ABOVE the coded version. Don't try to solve in your head. One line of writing saves 30 seconds of confusion.

Data Sufficiency

3-5 questions. RBI loves these. Check if question can be answered, not what the answer is.
The 5 Answer Options

Memorize the standard option pattern

  1. Statement I alone is sufficient — but Statement II alone is not
  2. Statement II alone is sufficient — but Statement I alone is not
  3. Either Statement I or II alone is sufficient — each independently gives answer
  4. Both I and II together are sufficient — but neither alone is enough
  5. Both I and II together are NOT sufficient — even combined, answer cannot be found
TRICK: Ask "Can I answer the question with this statement ALONE?" Don't solve fully. Just check if unique answer is possible. If Statement I gives a definite value → mark I sufficient. If II also gives definite value → answer is "Either."
Example — Ordering
Question: Who is tallest among A, B, C?
I. A is taller than B. II. C is taller than A.
I alone: A > B. Don't know about C. Not sufficient. II alone: C > A. Don't know about B. Not sufficient. Both: C > A > B. C is tallest. Sufficient together.
Answer: Both I and II together are sufficient (Option 4)
Example — Either Case
What is the code for 'book'?
I. 'read a book' is coded as 'li si da'
II. 'book is good' is coded as 'da ro pa'
I alone: 'book' could be 'li', 'si', or 'da'. Not sufficient alone. Wait — actually need to compare. I gives possibilities. II gives 'da' for 'book' (common word? No, need to check). Actually with only I: 3 possibilities. With only II: 3 possibilities. Neither alone. Both: 'book' is common between phrases. In I: 'li si da'. In II: 'da ro pa'. Common code = 'da'. So 'book' = 'da'.
Answer: Both together (Option 4). Wait — if 'book' was uniquely identifiable in I alone, answer would be different. Here we need both.

Coding-Decoding — All Variations

3-5 questions. Pattern types repeat. Know all 6.
1. Letter Shift
Each letter +1 or +2 or +3
CAT → DBU (+1 each)
2. Reverse then Shift
Word reversed first, then shifted
CAT → TAC → UBD (+1)
3. Position Swap
First ↔ last, second ↔ second-last
HELLO → OELHL
4. Number Coding
Code uses numbers. A=1, B=2...
or reverse: Z=1, Y=2...
5. Word-to-Word
'Happy coding' = 'ro fi'
Common word analysis
6. Mixed Pattern
Vowels +2, Consonants -1
Or alternate shifts
The 3-Step Method

Works for 90% of coding questions

  1. Step 1: Check if letters are simply shifted. Count difference between first letters.
  2. Step 2: If shift doesn't work, check if word is reversed first.
  3. Step 3: If still no pattern, check position-based (A=1, B=2...) or mixed rules.
TRICK: Always test the SIMPLEST pattern first. 80% of RBI coding questions are simple letter shifts (+1, +2, -1, -2). Don't overthink.
Example — Word-to-Word (RBI Favorite)
'good morning' is coded as 'la pa'
'morning walk' is coded as 'pa fi'
What is code for 'good'?
Common word: 'morning' appears in both Common code: 'pa' appears in both So 'morning' = 'pa' In first: 'good morning' = 'la pa' → 'good' = 'la'
Answer: 'la'
Example — Position Swap
In a code, FRIEND is written as DNEIRF. How is GARDEN written?
FRIEND reversed = DNEIRF Pattern: Reverse the word GARDEN reversed = NEDRAG
Answer: NEDRAG

Blood Relations + Family Tree

2-3 questions. Shorthand method saves 60% time.
Father's father
Grandfather (male, 2 levels up)
Father's mother
Grandmother (female, 2 levels up)
Sister's son
Nephew (male, 1 level down)
Brother's daughter
Niece (female, 1 level down)
Mother's brother
Maternal Uncle
Father's sister
Paternal Aunt
Wife's father
Father-in-law
Husband's mother
Mother-in-law
The Shorthand Method

Write relationships, don't draw full tree

  1. Use ↑ for parent (father/mother), ↓ for child (son/daughter)
  2. Use = for spouse (wife/husband)
  3. Use ↔ for sibling (brother/sister)
  4. Track gender with M/F subscripts
SHORTCUT: When they say "pointing to X, Y said..." — the person Y is talking ABOUT is X. Y's statement describes X's relationship to someone. Start from the known person.
Example — Complex Statement
Pointing to a man, a woman said, "His mother is the only daughter of my mother." How is the woman related to the man?
"Only daughter of my mother" = the woman herself (she's the only daughter) So "His mother" = the woman The woman is the man's mother
Answer: Mother
Example — Maternal/Paternal
A is B's sister. C is B's mother. D is C's father. E is D's mother. How is A related to D?
D is C's father, C is B's mother → D is B's grandfather A is B's sister → A is also D's granddaughter
Answer: Granddaughter

Direction & Distance

2-3 questions. Coordinate method + turn logic.
The Coordinate Method

Start at (0,0). North = +Y, East = +X.

  1. 10 m North → (0, +10)
  2. 5 m East → (+5, +10)
  3. 10 m South → (+5, 0)
  4. 5 m West → (0, 0) → Back to start!
SHORTCUT: Opposite directions CANCEL. N+S cancel. E+W cancel. Only NET movement matters. Distance from start = sqrt(X² + Y²).
Turn Directions

Clock face method for turns

  1. Imagine a clock: 12=North, 3=East, 6=South, 9=West
  2. Turning LEFT = moving counter-clockwise
  3. Turning RIGHT = moving clockwise
  4. Facing North, turn left → West. Turn right → East.
  5. Turn 180° = opposite direction
SHORTCUT: After LEFT turn: N→W, W→S, S→E, E→N. After RIGHT turn: N→E, E→S, S→W, W→N. Just memorize one direction, the rest follow.
Example — With Turns
A man walks 10 km North, turns left and walks 5 km, turns left and walks 10 km, turns left and walks 5 km. Where is he?
Start: (0,0), facing North Walk 10 km N → (0, 10), turn left (now facing West) Walk 5 km W → (-5, 10), turn left (now facing South) Walk 10 km S → (-5, 0), turn left (now facing East) Walk 5 km E → (0, 0)
Answer: Back at starting point

Order & Ranking

2-3 questions. 3 formulas cover all cases.
The 3 Formulas

Total, Position from one side, Position from other side

  1. Total = (Position from left) + (Position from right) - 1
  2. Position from right = Total - Position from left + 1
  3. People between A and B = (Total) - (Pos A + Pos B) when counted from same side
TRICK: Draw a line with positions. Mark left and right. If A is 5th from left and B is 8th from left, people between them = 8 - 5 - 1 = 2. Subtract 1 because you don't count either A or B.
Example — From Both Sides
In a row of 40 students, A is 15th from left and B is 20th from right. How many students are between A and B?
B's position from left = 40 - 20 + 1 = 21st from left A is 15th from left, B is 21st from left Between them = 21 - 15 - 1 = 5 students
Answer: 5 students
Example — Interchanged Positions
In a row, Ram is 15th from left. When he interchanges position with Shyam, Ram becomes 20th from left. What was Shyam's original position from left?
Ram moved from 15th to 20th → moved 5 positions right Shyam must have been at 20th originally (they swapped) So Shyam was 20th from left originally
Answer: 20th from left

Statement — Assumption & Conclusion

2-3 questions. Logical reasoning, not verbal.
Assumption vs Conclusion

Assumption = unstated premise. Conclusion = stated deduction.

  1. Assumption: What MUST be true for the statement to make sense? Something NOT stated but implied.
  2. Conclusion: What DEFINITELY follows from the statement? A logical deduction.
  3. Rule for Assumption: If statement is "X is announced", assumption = "X was not already known" or "X will be implemented."
  4. Rule for Conclusion: Only what is DIRECTLY implied. Don't bring outside knowledge.
SHORTCUT: For assumptions, ask "Would the statement still make sense if this assumption were FALSE?" If NO, then it's a valid assumption.
Example — Assumption
Statement: "Please close the door."
Assumptions: I. The door is open. II. The speaker wants privacy.
I. The door is open → MUST be true (otherwise why ask to close?). VALID. II. The speaker wants privacy → Not necessarily. Could be for temperature, noise, etc. NOT VALID.
Answer: Only I is implicit

Alpha-Numeric Series

1-2 questions. Letter position values + pattern recognition.
Letter Position Values

Memorize A=1 to E=5, J=10, O=15, T=20, Z=26. Rest you can count.

  1. A=1, B=2, C=3, D=4, E=5
  2. F=6, G=7, H=8, I=9, J=10
  3. K=11, L=12, M=13, N=14, O=15
  4. P=16, Q=17, R=18, S=19, T=20
  5. U=21, V=22, W=23, X=24, Y=25, Z=26
SHORTCUT: Reverse positions: A=26 (from Z), B=25, C=24... If question uses reverse alphabetical order, subtract from 27. (A=1 from start = 26 from end = 27-1).
Example — Letter Series
Find the next letter: B, E, H, K, ?
B(2), E(5), H(8), K(11) Differences: +3, +3, +3, +3 Next = 11 + 3 = 14 = N
Answer: N
Example — Number-Letter Mixed
Find the missing term: 2B, 4D, 8F, 16H, ?
Numbers: 2, 4, 8, 16 → doubling. Next = 32. Letters: B(2), D(4), F(6), H(8) → +2 each. Next = J(10).
Answer: 32J

Puzzles & Seating Arrangement

15-20 questions. The biggest section. All variations covered.
Linear (Single Row)
People in a line. Some face North, some South. Draw boxes _ _ _ _ _. Place fixed positions first.
Linear (Double Row)
Two parallel rows facing each other. Row 1 faces South, Row 2 faces North. Track who faces whom.
Circular
Round table. All face center OR all face outward. "Immediate left" depends on facing direction.
Rectangular
People at corners + middle of sides. 8 people typically. Corner people have 2 neighbors.
Floor-Based
People on different floors of a building. Usually with flats (1-2 flats per floor).
Day/Month Based
People + dates + months. Create a grid: People x (Date/Month/Other attribute).
Comparison
A is taller than B but shorter than C. Rank order puzzles. Draw vertical ranking.
Box/Stack
Boxes stacked one above another. Color/number/item in each box.
The Universal Puzzle Method

Works for ALL puzzle types

  1. Read ALL clues first. Don't solve as you read.
  2. Find the "fixed position" clue. "A sits at an extreme end." "B lives on top floor."
  3. Place the most constrained person/thing first. The one mentioned in most clues.
  4. Use elimination. If A is NOT on floor 1, and NOT on floor 5, and floors 2,3 are taken → A is on floor 4.
  5. Mark possibilities. If unsure between two positions, write both lightly and eliminate as you go.
GOLDEN RULE: If a puzzle takes >3 minutes, MARK it and move on. Come back only if time permits. 1 puzzle = 5 questions. Losing 5 marks AND 5 minutes is worse than skipping.
Circular Seating — Direction Trick

"Immediate left" changes based on facing direction

  1. If ALL face CENTER: "Immediate left" = clockwise. "Immediate right" = counter-clockwise.
  2. If ALL face OUTWARD: "Immediate left" = counter-clockwise. "Immediate right" = clockwise.
  3. Remember: When you face center, your left hand points clockwise.
TRICK: Imagine yourself sitting at the table. If facing center, left = clockwise. If facing wall (outward), left = counter-clockwise. Don't memorize, imagine.
Example — Floor-Based Puzzle
7 people A,B,C,D,E,F,G live on 7 floors (1-7, 1=ground). A lives on floor 4. B lives immediately above C. D lives immediately below E. F lives on top floor. G does not live on floor 1 or 7. B does not live on floor 2. Where does G live?
Fixed: A=4, F=7 B immediately above C: possible pairs (2,1), (3,2), (5,4)-no A=4, (6,5), (7,6)-no F=7. So (2,1), (3,2), (6,5). B not on 2 → eliminate (2,1). So B,C = (3,2) or (6,5). D immediately below E: pairs (1,2), (2,3), (5,6), (6,7)-no F=7. If B,C = (3,2): then D,E = (5,6). Floors used: 1,2,3,4,5,6,7. G=1. But G not on 1. Contradiction! If B,C = (6,5): then D,E = (1,2) or (2,3). G must be on 3 or 2 (whichever not used). G not 1 or 7. Works. If D,E=(1,2): floors used 1,2,4,5,6,7. G=3. If D,E=(2,3): floors used 2,3,4,5,6,7. G=1. But G not on 1. So D,E=(1,2), G=3.
Answer: G lives on floor 3

Input-Output

2-3 questions. Machine rearrangement. Pattern recognition.
Common Patterns

6 patterns cover 90% of input-output questions

  1. Ascending order: Arrange words alphabetically OR numbers increasingly. One element per step.
  2. Descending order: Reverse alphabetical or decreasing numbers.
  3. Alternating: Step 1: smallest number. Step 2: first alphabet word. Step 3: next number. Step 4: next word...
  4. Vowel-Consonant: Words starting with vowels first, then consonants.
  5. Odd-Even: Odd numbers first (ascending), then even numbers (ascending).
  6. Interchange: First and last interchange, then second and second-last, etc.
TRICK: Compare Input → Step 1. What changed? If only ONE element moved = single element per step. If MULTIPLE changed = batch rearrangement. Most RBI questions are single element per step.
Example — Alternating Pattern
Input: 24 boy 11 63 girl 42 apple 75
Step 1: 11 24 boy 63 girl 42 apple 75
Step 2: 11 apple 24 boy 63 girl 42 75
Step 3: 11 apple 24 42 boy 63 girl 75
Find Step 5.
Step 1: smallest number (11) moved to first Step 2: first alphabet word (apple) moved to second Step 3: next smallest number (24→42? No, 24 was already in place. Next smallest is 42) → wait, 24 was at position 2. Next number after 11 is 24... already there. Then 42. Actually pattern: Number (ascending), Word (alphabetical), Number, Word... Step 4: Next word alphabetically = boy → 11 apple 24 boy 42 63 girl 75 Step 5: Next number = 63 → 11 apple 24 boy 42 63 girl 75
Answer: Step 5 = 11 apple 24 boy 42 63 girl 75
