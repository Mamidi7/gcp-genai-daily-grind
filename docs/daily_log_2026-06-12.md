## Interview Prep

- Snowflake ingestion interview framing:
  - **30s answer**: "Batch ingestion in Snowflake uses named stages (internal or cloud), file formats, and COPY INTO. The pipeline is: stage files arrive → optionally validated with VALIDATION_MODE → loaded with COPY INTO. Snowpipe extends this for near-real-time via event notifications."
  - **90s STAR**: Situation = daily CSV deliveries from a legacy system needed loading into Snowflake. Task = build a reliable ingestion pipeline. Action = created named stage with file format, used COPY INTO with ON_ERROR=CONTINUE to handle bad rows without blocking good data, set up Snowpipe for fresh files. Result = 99.9% load success rate, bad rows quarantined in a reject table for reprocessing.
  - **Gotcha to mention**: "ON_ERROR=CONTINUE vs SKIP_FILE — CONTINUE logs bad rows and continues with the rest of the file; SKIP_FILE drops the entire file. For production ETL, I'd use CONTINUE + a reject table pattern so we can trace what failed."
