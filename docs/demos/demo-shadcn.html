Flask → Cloud Run

From a Python Flask app to a production container running on Cloud Run. No Kubernetes, no complex infra.

Architecture

Browser → Cloud Run → Gunicorn → Flask — four layers, one request path.

Layer 1

Browser / Client

Sends HTTP request. Could be curl, web app, mobile client, or another service calling your API.

Layer 2

Cloud Run

Managed compute. Receives request, starts container if cold, routes to Gunicorn on port 8080.

Layer 3

Gunicorn WSGI

Production WSGI server. Manages worker processes, handles concurrent requests.

Code

The four files that make it work.

main.py

Flask app with health endpoint

# main.py from flask import Flask, jsonify app = Flask(__name__) @app.route("/") def hello(): return {"message": "Hello from Flask on Cloud Run!"} @app.route("/health") def health(): return {"status": "healthy"} if __name__ == "__main__": app.run(host="0.0.0.0", port=8080)

requirements.txt

flask==3.1.0
gunicorn==23.0.0

Why Gunicorn?

Flask's dev server is single-threaded. Under concurrent load on Cloud Run, it crashes. Gunicorn manages worker processes and handles concurrent requests properly.
